1 Information We Collect

A. Personal Information

We do NOT collect, store, or transmit any personally identifiable information (PII). You do not need to create an account or provide any personal details to use BookmarkHero.

B. Bookmark Data

BookmarkHero requests the bookmarks permission to read, organize, and manage your browser's bookmarks.

All analysis (detecting dead links, finding duplicates, identifying empty folders) is performed entirely locally within your browser. None of your bookmark data is uploaded anywhere.

C. Network Requests for Dead Link Detection

To verify if a saved bookmark is still active, the extension makes brief HTTP network requests to the URLs stored in your bookmarks. These requests:

• Use HTTP HEAD or truncated GET requests strictly to check server availability.

• Do not attach any user context, cookies, or personal tracking identifiers.

• Are initiated directly from your browser — no proxy or intermediary server is involved.

2 Permissions Used and Why

BookmarkHero requests the minimum permissions necessary to function:

Permission Why It's Needed
bookmarks Read your bookmark tree, detect duplicates, clean empty folders, and remove dead links.
storage Save your extension preferences (e.g., scanner settings, AI provider) and operation logs (undo history) locally on your device only.
sidePanel Provide a seamless side-panel UI for quick access to all tools.
notifications Show desktop notifications when AI auto-classification suggests moving a newly saved bookmark.
<all_urls> Allow the background service worker to make HTTP requests to your bookmarked websites for dead link detection. No user data is transmitted.

3 AI Features & Data Privacy

Gemini API (Cloud)

If you choose to use the Gemini cloud AI feature, the title and URL of the bookmark being classified are sent to Google's Gemini API to generate a folder suggestion. This is governed by Google's Privacy Policy. Your API key is stored locally and never transmitted to our servers.

Ollama (Local)

When using the Ollama local AI mode, bookmark data is sent only to your own machine's local Ollama service. No data leaves your computer whatsoever.
